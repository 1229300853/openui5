The Basics!

You can use OpenUI5 in any HTML file by including the following bootstrap script tag (make sure that you have put the UI5 runtime download in your application folder, if you reference it like in this example): 

<script id="sap-ui-bootstrap"
   src="resources/sap-ui-core.js"
   data-sap-ui-theme="sap_bluecrystal"
   data-sap-ui-libs="sap.m">
</script>

The parameters used above determine which resources are loaded apart from the UI5 core. For instance, the data-sap-ui-theme parameter determines which of the available themes the application will be using, and the data-sap-ui-libs parameter defines which control libraries you want to use.

A very simple application would look like this:

<!DOCTYPE html>
<html><head>
<meta http-equiv='X-UA-Compatible' content='IE=edge' />
<title>Hello World</title>

<script id='sap-ui-bootstrap'
    src='resources/sap-ui-core.js'
    data-sap-ui-theme='sap_bluecrystal'
    data-sap-ui-libs='sap.m'
    data-sap-ui-compatVersion='edge'
    data-sap-ui-preload='async>

</script>

<script>
         sap.ui.getCore().attachInit(function () {
             jQuery("#content").html("Hello World - UI5 is ready");
         });
</script>

</head>
<body class='sapUiBody'>
    <div id='content'></div>
</body>
</html>

In this step we load the OpenUI5 framework from our local webserver and initialize the core modules with the following configuration options:

  • We specify the required UI library “sap.m” containing the UI controls we need for this tutorial.
  • The OpenUI5 controls support different themes, we choose “sap_bluecrystal” as our default theme.
  • To make use of the most recent functionality of OpenUI5 we define the compatibility version as “edge”
  • We configure the process of “bootstrapping” to run asynchronously

The src attribute of the first <script> tag tells the browser where to find the OpenUI5 core library – it initializes the SAPUI5 runtime and loads additional resources such as the libraries specified in the “data-sap-ui-libs” attribute. When all resources and libraries are loaded SAPUI5 is ready. Then the “init” event of the core is fired and all registered handlers are executed. This process of loading and initializing SAPUI5 is called bootstrapping. There are two versions of the SAPUI5 core: one with jQuery and one without jQuery. In our case we load the core bundled with jQuery. In our example the resources are loaded asynchronously. This means that the resources can be loaded simultaneously in the background for performance reasons. Once the SAPUI5 core and the corresponding resources have been loaded we can start using it. Therefore, the library fires the init event to signal that the library is ready. It is a good practice to listen for this event in order to trigger your application logic only after the event has been fired. In the example above we get a reference to the SAPUI5 core by calling sap.ui.getCore()and register an anonymous callback function by calling attachInit(…)on the core. In SAPUI5 callback functions are often referred to as handlers, listener functions, or simply listeners. The core is a Singleton and can be accessed from anywhere in the code. Our anonymous callback function is executed when the bootstrap of OpenUI5 is finished and displays a native JavaScript alert.

Using OpenUI5 Controls

Now it is time to build our first little UI by replacing the “Hello World” text in the HTML body by the OpenUI5 control sap.m.Text. For the beginning we use the JavaScript control interface to setup the UI. The control instance is then "placed" into the HTML body. Don’t forget to remove the line starting with "jQuery("#content")... ".

Here's a preview:

And here’s the code for this step:

<!DOCTYPE html>
<html><head>
<meta http-equiv='X-UA-Compatible' content='IE=edge' />
<title>Hello World</title>

<script id='sap-ui-bootstrap'
    src='resources/sap-ui-core.js'
    data-sap-ui-theme='sap_bluecrystal'
    data-sap-ui-libs='sap.m'
    data-sap-ui-compatVersion='edge'
    data-sap-ui-preload='async>
</script>

<script>
         sap.ui.getCore().attachInit(function () {
             	new sap.m.Text({
			 text: "Hello World"
		}).placeAt("content");
         });
</script>

</head>
<body class='sapUiBody'>
    <div id='content'></div>
</body>
</html>

Instead of using native JavaScript to change the content of the body tag, we want to use a simple SAPUI5 control. Controls are used to define appearance and behavior of parts of the screen. In the example above the content of the init event handler is now replaced with an OpenUI5 text control. The name of the control is prefixed by the namespace of its control library “sap.m” and the options are passed to the constructor with a JavaScript object. For our control we set the “text” property to the value “Hello World”. We concatenate the constructor call of the control with the standard method “placeAt” that is used to position OpenUI5 controls inside a node of the DOM. As the target node we use the body tag of the HTML document with the id "content". The class “sapUiBody” adds additional theme dependent styles for displaying OpenUI5 apps. All Controls of OpenUI have a fixed set of properties and methods to interact with, the public API of the control. You can it look up in the API reference that is linked below.

    Putting all our UI into the index.html file will very soon result in a messy setup and there is quite a bit of work ahead of us. So let’s do a first for modularization by putting the sap.m.Text control into a dedicated “view”. OpenUI5 supports multiple view types (XML, HTML, JavaScript). We choose XML as this produces the most readable code and will force us separate the view declaration from the controller logic. Yet the look of our UI will not change.

    The outcome of this step will still look like this: (preview)

    We create a new file for our XML view. The root node of the XML structure is this view. Here we reference the default namespace "sap.m" where the majority of our UI assets are located. We define an additional "sap.ui.core.mvc" namespace where the OpenUI5 views and all other Model-View-Controller (MVC) assets are located with an alias "mvc". The view code will look like this:

    <mvc:View
	xmlns="sap.m"
	xmlns:mvc="sap.ui.core.mvc">
	<Text text="Hello World"/>
</mvc:View>

    Inside the view tag we add the declaratively definition of our text control with the same properties as in the previous step. The XML tags are mapped to controls and the attributes are mapped to the properties of the control. In OpenUI5 each control has its own id. In the XML View above we did not specify an id attribute and therefore the OpenUI5 runtime generates an own id and adds it to the control. However, it is a good practive to set the ids of controls explicitely so that controls can be identified easily.

    Next, we will change our application code, which will be placed in the index.html again:

    <!DOCTYPE html>
<html><head>
<meta http-equiv='X-UA-Compatible' content='IE=edge' />
<title>Hello World</title>

<script id='sap-ui-bootstrap'
    src='resources/sap-ui-core.js'
    data-sap-ui-theme='sap_bluecrystal'
    data-sap-ui-libs='sap.m'
    data-sap-ui-compatVersion='edge'
    data-sap-ui-preload='async>
</script>

<script>
   sap.ui.getCore().attachInit(function () {
        new sap.ui.xmlview({
		viewName: "sap.ui.demo.wt.App"
	}).placeAt("content");
   });
</script>

</head>
<body class='sapUiBody'>
    <div id='content'></div>
</body>
</html>

    We replace the instantiation of the sap.m.Text control by our new “App” XML view. The view is created by a factory function of OpenUI5 which makes sure that the view is correctly configured. The name is prefixed with the namespace “sap.ui.demo.wt.view” in order to uniquely identify this resource. Furthermore, we tell the OpenUI5 core that resources in the “sap.ui.demo.wt” namespace are located in the same folder as the index.html. This is important and necessary for apps to run in some containers.

    Controllers

    In this step we replace the text with a button and show the "Hello World" message once the button gets pressed. The handling of this event will be implemented on the controller of the view.

    Preview:

    Here is the new code for our view:

    <mvc:View
	controllerName="sap.ui.demo.wt.controller.App"
	xmlns="sap.m"
	xmlns:mvc="sap.ui.core.mvc">
	<Button
		text="Say Hello"
		press="onShowHello"/>
</mvc:View>

    We replace the text control with a button that carries the text “Say Hello” and calls the "onShowHello" function when being pressed. We also have to state the name of the controller that holds the "onShowHello" function. We create a new file - the controller for our view, which we will call App.controller.js. 

    sap.ui.define([
	"sap/ui/core/mvc/Controller"
], function (Controller) {
	"use strict";

	return Controller.extend("sap.ui.demo.wt.App", {

		onShowHello : function () {
			// show a native JavaScript alert
			alert("Hello World");
		}
	});

});

    We define the App controller in its own file by extending the "Controller" object of the OpenUI5 core. For the beginning it holds only a single function called "onShowHello" that handles the pressing of the button by showing an alert.

    Conventions

    • Controller names are capitalized
    • Controller carry the same name as related view (if there is a 1:1 relationship)
    • Event handlers are prefixed with “on”
    • Controllers always end with *.controller.js

    <mvc:View xmlns:core="sap.ui.core" xmlns:mvc="sap.ui.core.mvc" xmlns="sap.m" controllerName="getStarted" xmlns:html="http://www.w3.org/1999/xhtml"> Get started with MVC<html:br/> <Panel text="Hello World"> <Button text="{/actionName}" press="doSomething"/> <html:table border="5"> <html:tr> <html:td>An</html:td> <html:td>HTML Table</html:td> </html:tr> </html:table> </Panel> </mvc:View>

    In an application structure, this code would e.g. go into a file named getStarted.view.xml. Let’s take a closer look at the Button control within this view. It does not display a simple text any more, instead it gets its new value from a model via data binding. You can use the notation in curly braces to reference a particular node in a model. In our example, we will be using a simple JSON model, but we’ll talk more about this in a minute. First, note how the control has a press handler attached – obviously, it is trying to invoke a function called doSomething. We'll define this function in our controller belonging to this view, and so the code for this controller could look like this: 

    sap.ui.controller("getStarted", {
	// implement an event handler in the Controller
	doSomething: function() {
		alert("Hello World!");
	}
});

    Again, this code would go into yet another separate file, which we could call getStarted.controller.js. As a last step, now we'll modify our actual page in order to display our new view instead of a simple button, and also create the model for our data binding:

    <!DOCTYPE html>
<html>
  <head>
    <meta http-equiv='X-UA-Compatible' content='IE=edge' />
	<title>UI5 MVC with XmlView</title>
	<script id='sap-ui-bootstrap' type='text/javascript'
		src='https://openui5.hana.ondemand.com/resources/sap-ui-core.js'
		data-sap-ui-theme='sap_bluecrystal'
		data-sap-ui-libs='sap.m'>
	</script>
	<script>
	/*** THIS IS THE "APPLICATION" CODE ***/
		// create some dummy JSON data
		var data = {
			actionName: "Say Hello"
		};
		// instantiate the View
		var myView = sap.ui.xmlview(“getStarted”);
		// create a Model and assign it to the View
		var oModel = new sap.ui.model.json.JSONModel();
		oModel.setData(data);
		myView.setModel(oModel);
		// put the View onto the screen
		myView.placeAt('content');
	</script>
	</head>
	<body class='sapUiBody'>
		<div id="content"></div>
	</body>
</html>

    As you can see, here we've created a small JSON model (which would again come from a separate resource in a real application) with one node inside it, which happens to be the attribute our Button text is bound against. Next, we're instantiating our view and then placing it within a div tag that has the id “content” on our page, using the placeAt method you already know from earlier.

    Wiring the Elements – Data Binding in UI5

    Databinding in UI5 is simple yet powerful - check out a more complex control like our Table bound to a JSON model here:

    JSBin sample - UI5 table example with local JSON model

    (As these examples are now getting more complex, open the JSBin code in a separate window - that way there's more space available, and you can start modifying straight away while reading the code explanations below.)

    Like in the previous example, this example also features a JSON model, but UI5 also supports other models as well of course – you can easily connect your application to an OData service, for instance.

    We start by creating an instance of a Table control:

    var oTable = new sap.ui.table.Table({editable:true});

    Now let's create the columns. For these, we first need to decide what the columns are supposed to contain. Let’s say, my first column should contain the last names from the model entries. For this content, we'll use a TextView control which will contain the model data, so let's create an instance like this:

    var oControl = new sap.m.Text({text:"{lastName}"});

    Note that this control is bound to a particular attribute from the model, namely lastName, but not to a particular node in the model. This is of course intentional, as we'll see in a minute. To make our table repeat the control in the first column as often as there are entries in the model, we just pass the control as a template into our table column instance, like this:

    oTable.addColumn(new sap.ui.table.Column({
	label: new sap.m.Label({text: "Last Name"}), 
	template: oControl, // our previously created TextView control
	sortProperty: "lastName", // with this property, sorting on a table column is activated
	filterProperty: "lastName", // with this property, filtering on values within a table column is activated
	width: "100px"
}));

    We define the other columns likewise and then we just need to create our JSON model from our sample data and bind the table to it: 

    var oModel = new sap.ui.model.json.JSONModel();
oModel.setData({modelData: aData}); //aData has already been defined in the application code
oTable.setModel(oModel);

    The rows in the table are now automatically generated from the model.

    It's all about controls.

    So you’re fine with the basics, but want to know more about UI5 controls, which ones are out there already, and how you can create your own?

    You can either dive right into our documentation - especially our Explored App - to find out about existing controls and how to use them here, or have a look at an example on how to write your own controls and use them in your applications.

    Again, while reading the code explanations below, we suggest you open the complete JSBin code example in a separate window, as space within this page is rather limited for more complex examples:
    JSBin sample - Create your own UI5 control

    Let's start with a very simple control here, just drawing a square div on our screen. As we don’t yet have any similar control available, we will extend the base Control delivered with UI5. We'll call the new Control type "my.Square" and let it inherit from sap.ui.core.Control.

    sap.ui.core.Control.extend("my.Square", {

    This will provide us with a lot of basic functionality required in all UI5 controls, e.g. generated getters and setters for any attributes we define in the control metadata as well as everything required to connect this control to our databinding. (Note: We could also extend any other existing control from any UI5 library, but let’s keep it simple for now.)

    We'll define the control metadata, defining some properties for our new control: 

    	metadata : {
		properties : { // setter and getter are created behind the scenes, incl. data binding and type validation
			"text" :  "string", // in simple cases, just define the type
			"size" : {
				type:  "sap.ui.core.CSSSize", 
				defaultValue:  "200px"
		} // you can also give a default value and more
	}},

    Now that we know which properties our control has, let’s create the renderer. Each control has a renderer function which defines what is displayed on screen. The renderer method is responsible for creating the HTML structure that makes up the control. It is different from the other methods, as it is a static one, so the this keyword is not available. Instead, a control instance and a RenderManager instance are given to the method, as shown below.

    	// the part creating the HTML:
	 renderer :  function(oRM, oControl) { // static function, so use the given "oControl" instance instead of "this"
		oRM.write("<div"); 
		oRM.writeControlData(oControl); // writes the Control ID and enables event handling - important!
		oRM.addStyle("width", oControl.getSize()); // write the Control property size
		oRM.addStyle("height", oControl.getSize());
		oRM.writeStyles();
		oRM.addClass("mySquare"); // add a CSS class for styles common to all control instances
		oRM.writeClasses(); // this call writes the above class plus enables support for Square.addStyleClass(...)
		oRM.write(">");
		oRM.writeEscaped(oControl.getText()); // write another Control property, with XSS protection
		oRM.write("</div>");
	},

    Finally, let's define an event handler for our control which will be called whenever someone clicks on our control:

    	// an event handler:
	onclick : function(evt) {   // is called when the Control's area is clicked - no event registration required
		alert("Control clicked! Text of the Control is:\n" + this.getText());
	}
}); // this line closes the curly brace from the very 1st code line

    Now we've done all of that, we can create instances from your new control and place them on our screen, e.g. like this: 

    var myControl = new my.Square({text:"Hello", size: "100px"});
myControl.placeAt("content");

// ok, add another instance...:
new my.Square({text:"Me too!", size: "80px"}).placeAt("content");
