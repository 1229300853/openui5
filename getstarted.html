The Basics!

You can use OpenUI5 in any HTML file by including the following bootstrap script tag (make sure that you have put the UI5 runtime download in your application folder, if you reference it like in this example): 

<script id="sap-ui-bootstrap"
   src="resources/sap-ui-core.js"
   data-sap-ui-theme="sap_bluecrystal"
   data-sap-ui-libs="sap.m">
</script>

The parameters used above determine which resources are loaded apart from the UI5 core. For instance, the data-sap-ui-theme parameter determines which of the available themes the application will be using, and the data-sap-ui-libs parameter defines which control libraries you want to use.

A very simple application would look like this:

<!DOCTYPE html>
<html><head>
<meta http-equiv='X-UA-Compatible' content='IE=edge' />
<title>Hello World</title>

<script id='sap-ui-bootstrap'
    src='resources/sap-ui-core.js'
    data-sap-ui-theme='sap_bluecrystal'
    data-sap-ui-libs='sap.m'>
</script>

<script>
    var btn = new sap.m.Button({
        text:'Press me!',
        press: function() {
            alert("Hello!")
        }
    });
    btn.placeAt('content');
</script>

</head>
<body class='sapUiBody'>
    <div id='content'></div>
</body>
</html>

Let's take a closer look at the code above. Here we're basically instantiating a simple control using these lines:

var btn = new sap.m.Button({
	text:'Press me!',
	press: function() {
		alert("Hello!")
	}
    });

What we're actually doing is creating a new instance of a control named Button, which is part of our sap.m library. Within the constructor call, we're making use of the JSON notation to pass some initial values to the control right away, namely:

  • The value for the text displayed on the button
  • The handler attached to the button's "press" event, which in this case does nothing more than alerting "Hello" to the person who clicks the button.

Note that we could have achieved the same output by doing something like this:

var btn = new sap.m.Button();
btn.setText('Press me!');
btn.attachPress(function() {
			alert("Hello!");
		});

For all controls in UI5, there are getters and setters for each public attribute as well as attach and detach methods for events.

However, this code alone does not display anything on screen just yet. To accomplish this, we need one more line from above's example, namely this: 

btn.placeAt('content');

This line tells UI5 to render this control inside the div tag with the id "content" in the HTML code below these lines.

By using these few simple lines of code, we've just created this button: (click to see the alert)

Try it out yourself - feel free to modify and extend this example!

MVC in UI5

While the code above is indeed already a very small UI5 application, you would probably expect a more sophisticated way to get your content on screen. Let’s see how we could replace the simple control with a view, and find out how to use MVC in UI5. We'll start with an XML view, which you'd usually define as a separate file to your index page.

Here’s the code for the view:

<mvc:View xmlns:core="sap.ui.core" xmlns:mvc="sap.ui.core.mvc" xmlns="sap.m" 
		controllerName="getStarted" xmlns:html="http://www.w3.org/1999/xhtml">
	
	Get started with MVC<html:br/>
	<Panel text="Hello World">
		<Button text="{/actionName}" press="doSomething"/>
		<html:table border="5">
			<html:tr>
				<html:td>An</html:td>
				<html:td>HTML Table</html:td>
			</html:tr>
		</html:table>
	</Panel>
</mvc:View>

In an application structure, this code would e.g. go into a file named getStarted.view.xml. Let’s take a closer look at the Button control within this view. It does not display a simple text any more, instead it gets its new value from a model via data binding. You can use the notation in curly braces to reference a particular node in a model. In our example, we will be using a simple JSON model, but we’ll talk more about this in a minute. First, note how the control has a press handler attached – obviously, it is trying to invoke a function called doSomething. We'll define this function in our controller belonging to this view, and so the code for this controller could look like this: 

sap.ui.controller("getStarted", {
	// implement an event handler in the Controller
	doSomething: function() {
		alert("Hello World!");
	}
});

Again, this code would go into yet another separate file, which we could call getStarted.controller.js. As a last step, now we'll modify our actual page in order to display our new view instead of a simple button, and also create the model for our data binding:

<!DOCTYPE html>
<html>
  <head>
    <meta http-equiv='X-UA-Compatible' content='IE=edge' />
	<title>UI5 MVC with XmlView</title>
	<script id='sap-ui-bootstrap' type='text/javascript'
		src='https://openui5.hana.ondemand.com/resources/sap-ui-core.js'
		data-sap-ui-theme='sap_bluecrystal'
		data-sap-ui-libs='sap.m'>
	</script>
	<script>
	/*** THIS IS THE "APPLICATION" CODE ***/
		// create some dummy JSON data
		var data = {
			actionName: "Say Hello"
		};
		// instantiate the View
		var myView = sap.ui.xmlview(“getStarted”);
		// create a Model and assign it to the View
		var oModel = new sap.ui.model.json.JSONModel();
		oModel.setData(data);
		myView.setModel(oModel);
		// put the View onto the screen
		myView.placeAt('content');
	</script>
	</head>
	<body class='sapUiBody'>
		<div id="content"></div>
	</body>
</html>

As you can see, here we've created a small JSON model (which would again come from a separate resource in a real application) with one node inside it, which happens to be the attribute our Button text is bound against. Next, we're instantiating our view and then placing it within a div tag that has the id “content” on our page, using the placeAt method you already know from earlier.

Wiring the Elements – Data Binding in UI5

Databinding in UI5 is simple yet powerful - check out a more complex control like our Table bound to a JSON model here:

JSBin sample - UI5 table example with local JSON model

(As these examples are now getting more complex, open the JSBin code in a separate window - that way there's more space available, and you can start modifying straight away while reading the code explanations below.)

Like in the previous example, this example also features a JSON model, but UI5 also supports other models as well of course – you can easily connect your application to an OData service, for instance.

We start by creating an instance of a Table control:

var oTable = new sap.ui.table.Table({editable:true});

Now let's create the columns. For these, we first need to decide what the columns are supposed to contain. Let’s say, my first column should contain the last names from the model entries. For this content, we'll use a TextView control which will contain the model data, so let's create an instance like this:

var oControl = new sap.m.Text({text:"{lastName}"});

Note that this control is bound to a particular attribute from the model, namely lastName, but not to a particular node in the model. This is of course intentional, as we'll see in a minute. To make our table repeat the control in the first column as often as there are entries in the model, we just pass the control as a template into our table column instance, like this:

oTable.addColumn(new sap.ui.table.Column({
	label: new sap.m.Label({text: "Last Name"}), 
	template: oControl, // our previously created TextView control
	sortProperty: "lastName", // with this property, sorting on a table column is activated
	filterProperty: "lastName", // with this property, filtering on values within a table column is activated
	width: "100px"
}));

We define the other columns likewise and then we just need to create our JSON model from our sample data and bind the table to it: 

var oModel = new sap.ui.model.json.JSONModel();
oModel.setData({modelData: aData}); //aData has already been defined in the application code
oTable.setModel(oModel);

The rows in the table are now automatically generated from the model.

It's all about controls.

So you’re fine with the basics, but want to know more about UI5 controls, which ones are out there already, and how you can create your own?

You can either dive right into our documentation - especially our Explored App - to find out about existing controls and how to use them here, or have a look at an example on how to write your own controls and use them in your applications.

Again, while reading the code explanations below, we suggest you open the complete JSBin code example in a separate window, as space within this page is rather limited for more complex examples:
JSBin sample - Create your own UI5 control

Let's start with a very simple control here, just drawing a square div on our screen. As we don’t yet have any similar control available, we will extend the base Control delivered with UI5. We'll call the new Control type "my.Square" and let it inherit from sap.ui.core.Control.

sap.ui.core.Control.extend("my.Square", {

This will provide us with a lot of basic functionality required in all UI5 controls, e.g. generated getters and setters for any attributes we define in the control metadata as well as everything required to connect this control to our databinding. (Note: We could also extend any other existing control from any UI5 library, but let’s keep it simple for now.)

We'll define the control metadata, defining some properties for our new control: 

	metadata : {
		properties : { // setter and getter are created behind the scenes, incl. data binding and type validation
			"text" :  "string", // in simple cases, just define the type
			"size" : {
				type:  "sap.ui.core.CSSSize", 
				defaultValue:  "200px"
		} // you can also give a default value and more
	}},

Now that we know which properties our control has, let’s create the renderer. Each control has a renderer function which defines what is displayed on screen. The renderer method is responsible for creating the HTML structure that makes up the control. It is different from the other methods, as it is a static one, so the this keyword is not available. Instead, a control instance and a RenderManager instance are given to the method, as shown below.

	// the part creating the HTML:
	 renderer :  function(oRM, oControl) { // static function, so use the given "oControl" instance instead of "this"
		oRM.write("<div"); 
		oRM.writeControlData(oControl); // writes the Control ID and enables event handling - important!
		oRM.addStyle("width", oControl.getSize()); // write the Control property size
		oRM.addStyle("height", oControl.getSize());
		oRM.writeStyles();
		oRM.addClass("mySquare"); // add a CSS class for styles common to all control instances
		oRM.writeClasses(); // this call writes the above class plus enables support for Square.addStyleClass(...)
		oRM.write(">");
		oRM.writeEscaped(oControl.getText()); // write another Control property, with XSS protection
		oRM.write("</div>");
	},

Finally, let's define an event handler for our control which will be called whenever someone clicks on our control:

	// an event handler:
	onclick : function(evt) {   // is called when the Control's area is clicked - no event registration required
		alert("Control clicked! Text of the Control is:\n" + this.getText());
	}
}); // this line closes the curly brace from the very 1st code line

Now we've done all of that, we can create instances from your new control and place them on our screen, e.g. like this: 

var myControl = new my.Square({text:"Hello", size: "100px"});
myControl.placeAt("content");

// ok, add another instance...:
new my.Square({text:"Me too!", size: "80px"}).placeAt("content");
