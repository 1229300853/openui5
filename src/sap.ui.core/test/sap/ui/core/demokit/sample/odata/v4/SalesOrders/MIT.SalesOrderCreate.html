Manual Integration Test: Create Sales Order

DESCRIPTION

Test entity creation with the Sales Orders application.

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are no errors and warnings.

Testing with mock data

Start the Sales Orders application
URL (adjust host and port if necessary):
  • as explored application: http://<host>:<port>/sapui5-sdk-dist/explored.html#/sample/sap.ui.core.sample.odata.v4.SalesOrders/preview
  • local: http://<host>:<port>/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrders

Testing with backend

Configure your IDE to be able to execute the Sales Orders application against an ABAP backend system
Start the Sales Orders application
URL (adjust host and port if necessary):
  • local: http://<host>:<port>/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrders&realOData=true

REALIZE and CHECK

Create, modify and delete of an unsaved sales order

  1. After the application is started, click on the Create button in the Sales Orders table
  2. Check: The table below the Create New Sales Order dialog gets a new row with an empty Sales Order ID and the Note as displayed in the Create New Sales Order dialog
  3. Modify the content of the Note input field of the Create New Sales Order dialog
  4. Check: The Note of the first sales order (table below the dialog) is updated automatically after leaving the input field of the dialog.
  5. Click on OK in the Create New Sales Order dialog
  6. Select the newly created sales order
  7. Modify the content of the Note in the newly created sales order
  8. Click on the Delete button in the Sales Orders table
  9. Click on the Cancel button in the Sales Order Deletion dialog
  10. Check: The newly created sales order is still visible.
  11. Click on the Delete button in the Sales Orders table
  12. Click on the OK button in the Sales Order Deletion dialog
  13. Check: The newly created sales order is gone.

Create a sales order, save, modify again, save and delete

  1. Click on the Create button in the Sales Orders table
  2. Click on OK in the Create New Sales Order dialog
  3. Click on the Save button in the Sales Orders table
  4. Check: You get a success message for creating the sales order
  5. Click on OK in the Success dialog
  6. With backend only:
    1. Check: The Sales Order ID has been updated by the server
    2. For the newly created sales order enter a text in the Note field
    3. Click on Save button in the Sales Orders table
    4. Check: You get a success message for updating the sales order
    5. Click on OK in the Success dialog
  7. Select the saved sales order
  8. Click on Delete button in the Sales Orders table
  9. Click on the OK button in the Sales Order Deletion dialog
  10. Check: You get a success message for the deleted sales order
  11. Click on OK in the Success dialog
  12. Check: The newly created sales order is gone.

Create a sales order, save and refresh the sales orders

  1. Click on the Create button in the Sales Orders table
  2. Click on OK in the Create New Sales Order dialog
  3. Click on the Save button in the Sales Orders table
  4. Check: You get a success message for creating the sales order
  5. Click on OK in the Success dialog
  6. Click on Refresh button in the Sales Orders table
  7. Check: The first sales order is the same as the one from the initial application start
  8. Check (with backend only): The new sales order is inserted at the right position (considering sort and filter criterion); maybe you have to click on the More button of the Sales Orders table to see the new sales order

Create a sales order, refresh/filter the sales orders without saving

  1. Click on the Create button in the Sales Orders table
  2. Click on OK in the Create New Sales Order dialog
  3. Click on Refresh button in the Sales Orders table
  4. Check: You get a Warning about pending changes
  5. Click on the Cancel button
  6. Click on Refresh all button
  7. Check: You get a Warning about pending changes
  8. Click on the Cancel button
  9. With backend only:
    1. Enter a numerical value in the Filter by gross amount... search field above the Sales Orders table and press Enter
    2. Check: You get a Error because of unsaved changes
    3. Click on the Close button
